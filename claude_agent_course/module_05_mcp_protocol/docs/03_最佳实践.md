# Module 5.3 - MCP 最佳实践

> **学习目标**：掌握 MCP 服务设计与安全使用要点。

## 1. 工具要小而专

- 工具职责单一，输入输出清晰
- 避免“万能工具”，降低误调用风险

---

## 2. 明确参数与错误

- 参数类型清晰，必要参数显式声明
- 工具返回错误时设置 `is_error=True`

---

## 3. 控制权限

- 只开放必要工具
- 通过 `allowed_tools` 精细控制
- 敏感操作必须增加二次确认

---

## 4. 版本管理

- MCP Server 需要明确 `version`
- 重大变更应保留旧版本一段时间

---

## 5. 输出格式稳定

- 返回结构化 JSON 或固定格式文本
- 输出稳定，方便客户端消费与测试

---

## 6. 监控与审计

- 记录工具调用日志
- 对失败调用进行分析

---

## 7. 典型反模式

- 参数可选但实际必需
- 输出无结构、难以解析
- 工具名模糊难以理解

---
